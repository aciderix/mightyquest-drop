# Catalog — extracted facts, organized by content type

Everything here is regenerated by the scripts in `re/tools/` from the unpacked
client. Nothing is hand-guessed; each file notes how it was produced.

```
catalog/
├── pe/                 binary structure
│   ├── pe_layout.txt           sections, entropy, imported DLLs, entry point
│   ├── module_split.txt        game (Ubisoft) vs third-party library tally
│   └── game_source_tree.txt    ~4800 source files referenced in asserts/paths
├── classes/            C++ class catalog (from MSVC RTTI)
│   ├── classes_all.txt         119 demangled RTTI type names
│   ├── classes_by_group.txt    grouped: engine (Zouna _Z) / gameplay / CRT
│   └── vtables.txt             79 classes with a located vtable VA
├── network/            server protocol surface
│   ├── server_controllers.txt  30 RPC controllers (the API to reimplement)
│   ├── server_contracts.txt    105 .NET DTO type names (the wire types)
│   ├── ui_native_routes.txt    CEF UI <-> native command routes
│   └── boot_connection_symbols.txt  boot/connection state-machine symbols
├── config/             data-driven game configuration
│   ├── settings_json.txt        93 *Settings.json design-data files
│   └── vfs_config_paths.txt     CONFIG:> virtual-filesystem config paths
├── functions.csv       57,660 functions recovered by Ghidra (address,size,name)
└── pe/
    ├── function_classification.csv  every function -> GAME/THIRD_PARTY/UNKNOWN + module + source
    └── game_only_summary.txt        counts per bucket/module (the "game-only" view)
```

The function map is anchored by `re/tools/label_from_strings.py`, which xrefs the
build's leftover source-path assert strings to the functions that reference them
— 5,951 functions are tied to a specific `.cpp`/`.h` with certainty (no IAT
needed). The rest stay UNKNOWN until further analysis.

## How to regenerate
```bash
pip install pefile capstone unicorn
python3 re/tools/analyze.py     # pe/ + network/ contracts & controllers
python3 re/tools/rtti.py        # classes/
# config/* and the network/*routes are string sweeps; see docs/02-TOOLING.md
# functions.csv comes from the Ghidra headless run (docs/02-TOOLING.md)
```
